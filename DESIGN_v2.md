## 三、【省区看版】详细设计

### 3.1 结构

每个省份是一个可展开/折叠的卡片：

```
📍 浙江  ▼                                    [已铺开]  [华东]
    ┌────────────────────────────────────────────────────────┐
    │ 城市      状态        覆盖人群    启动日期    最新动态    │
    ├────────────────────────────────────────────────────────┤
    │ 杭州      ████████    职工+居民   2025-01-01  正式执行   │
    │ 宁波      ████████    职工+居民   2016-06-01  试点深化   │
    │ 温州      ████████    职工+居民   2020-09-01  正常运行   │
    │ 嘉兴      ████████    职工+居民   2020-09-01  正常运行   │
    │ 湖州      ████████    职工+居民   2024-10-01  新启动    │
    │ ...                                                   │
    └────────────────────────────────────────────────────────┘
```

### 3.2 4个维度（需要你确认）

我根据行业理解，建议城市显示的4个维度：

| 维度 | 说明 | 示例值 |
|------|------|--------|
| **维度1：状态** | 长护险实施状态 | 已启动 / 试点中 / 未启动 |
| **维度2：覆盖人群** | 覆盖哪些人群 | 职工 / 居民 / 职工+居民 / 城乡一体 |
| **维度3：启动日期** | 正式实施时间 | 2025-01-01 |
| **维度4：最新动态** | 最近一条关键信息 | 正式执行 / 新启动 / 费率调整 / 无 |

**但你需要确认**：这4个维度是否符合你的需求？或者你想换成其他维度？

其他可选维度：
- 筹资标准（每人每年多少钱）
- 待遇水平（报销比例/每日金额）
- 参保人数
- 基金支出
- 遴选状态（是否已遴选保险公司）

### 3.3 省份状态标签

- `已铺开`：所有地级市已启动（如浙江、山东）
- `部分启动`：部分城市已启动（如江苏）
- `规划中`：省级方案已出台，城市尚未全面启动
- `尚未启动`：无省级方案，无城市启动
- `试点深化`：首批试点城市，持续深化（如上海）

### 3.4 城市列表规则

- 每个省份列出**全部地级市**
- 已启动的：高亮显示，绿色进度条
- 未启动的：灰色显示，标注"未启动"
- 上海特殊：列出**16个区**作为"城市"
- 县级行政区：如有长护险的（如义乌），标注"(县级)"，放在所属地级市下方或独立列出
