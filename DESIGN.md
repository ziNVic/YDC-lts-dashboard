### 1.2 层级结构

```
省份（可展开/折叠）
  ├── 政策状态标签：[已出台方案] / [正在遴选] / [尚未启动] / [试点阶段]
  ├── 推进阶段： [省级方案已发布] / [部分城市已启动] / [全省铺开] / [规划中]
  ├── 城市列表（点击展开）
  │   ├── 市级行政区
  │   ├── 县级行政区（如义乌、桐庐等）
  │   └── 上海各区（黄浦、徐汇、浦东等16区）
  └── 关键数据：参保人数、基金规模、享受待遇人数
```

## 二、政策分类体系重构

### 2.1 两个大层级

```
政策层级
├── 第一层：国家级 + 省级（顶层设计）
│   ├── 国家法律/意见（如中办国办意见）
│   ├── 国家部委指导意见（如医保局项目目录）
│   └── 省级实施方案/指导意见
└── 第二层：城市级（落地执行）
    ├── 市级实施方案/试行办法
    ├── 县级细则
    └── 区县级操作规范
```

### 2.2 政策类型分类（供你确认/调整）

| 分类 | 说明 | 示例 |
|------|------|------|
| **正式政策** | 已发布实施的正式文件 | 实施方案、试行办法、管理办法 |
| **征求意见稿/草案** | 公开征求意见阶段 | 政策草案、征求意见稿 |
| **人大议案/政协提案** | 两会相关提案 | 委员提案、人大建议 |
| **基金监管** | 关于基金使用监管 | 基金管理办法、运行分析报告 |
| **处罚通报** | 违规处理结果 | 典型案例通报、飞行检查结果 |
| **失能评估** | 评估标准相关 | 评估管理办法、技术规范 |
| **服务规范** | 护理服务相关 | 服务项目目录、服务规范 |
| **定点机构** | 机构管理相关 | 定点管理实施细则、机构名单 |
| **动态解读** | 新闻稿、解读文章 | 医保局发布、媒体报道 |
| **培训/人才** | 护理员培训相关 | 长期照护师标准、培训方案 |

### 2.3 每个政策条目字段

```json
{
  "id": "p001",
  "title": "政策名称",
  "level": "national" | "provincial" | "city" | "county",
  "province": "所属省份",
  "city": "所属城市（可为空）",
  "category": "正式政策/草案/议案/基金监管/处罚通报/失能评估/服务规范/定点机构/动态解读/培训人才",
  "status": "已实施" | "征求意见中" | "已废止" | "草案阶段",
  "date": "发布日期",
  "source": "发布机构",
  "url": "原文链接",
  "summary": "摘要",
  "important": true/false
}
```

### 问题3：政策分类
上面我提出的10个分类是否合适？有没有遗漏或需要合并的？

### 问题4：数据更新频率
- 你希望这个平台的数据多久更新一次？（每天/每周/按需）
- 更新方式：我来搜索 → 更新数据 → 你重新生成网页？

### 问题5：超链接来源
- 政策原文链接：你是否有内部资源（如医保局官网链接库）？还是我来搜索匹配？
- 对于历史政策，很多链接可能已经失效，怎么处理？（标注"原文已失效"或链接到存档？）
