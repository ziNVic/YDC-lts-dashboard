## 一、总体定位

从"政策收集工具"升级为"长护险行业监测情报平台"。核心价值：
- 让使用者一眼看清全国28个省份的长护险推进节奏
- 追踪政策从"规划→实施→监管→执行"的完整生命周期
- 每份政策可直达原文，确保信息可追溯

## 三、五大视图详细说明

### 视图1：总览仪表盘

**用途**：快速掌握全国长护险推进全局

**内容组件**：
| 组件 | 内容 |
|------|------|
| 统计卡片区 | ① 已铺开省份数 ② 部分启动省份数 ③ 规划中省份数 ④ 尚未启动省份数 |
| 全国推进地图 | 各省以颜色区分：深绿=已铺开、浅绿=部分启动、橙色=规划中、灰色=尚未启动 |
| 最新动态时间线 | 按时间倒序排列政策发布、省份启动等关键事件 |
| 监管提醒 | 基金超支预警、检查通知等需要关注的事项 |
| 省份推进排行 | 按推进进度排序的省份TOP10 |

## 五、交互逻辑

### 5.1 省份卡片交互

| 操作 | 行为 |
|------|------|
| 点击省份卡片 | 展开/折叠城市列表 |
| 点击城市名 | 弹出详情弹窗（显示该城市相关政策、监管信息） |
| 区域筛选 | 左侧勾选区域，只显示对应省份 |
| 状态筛选 | 左侧勾选"已铺开/部分启动/规划中/尚未启动" |
| 搜索 | 顶部搜索框输入省份名或城市名，实时过滤 |

### 5.2 政策卡片交互

| 操作 | 行为 |
|------|------|
| 点击"🔗 查看原文" | 新标签页打开政策URL |
| URL失效时 | 链接可点击但标注"⚠️ 原文已失效"，点击后提示 |
| 点击层级筛选 | 只显示国家级/省级或城市级 |
| 点击分类筛选 | 只显示对应分类的政策 |
| 搜索 | 输入关键词，匹配标题、摘要、省份名 |

## 六、超链接处理方案

### 6.1 数据来源
- 政府官网（gov.cn、nhsa.gov.cn、各地医保局）
- 媒体报道（财新、新华社、证券时报等）
- 公开政策文件（PDF链接）

### 6.2 失效处理
- 数据文件中每个政策有 `url` 和 `urlStatus` 两个字段
- `urlStatus` 可选值：`有效` / `已失效` / `待验证`
- 前端展示：
  - 有效：`🔗 查看原文`（蓝色可点击）
  - 已失效：`⚠️ 原文已失效`（灰色，点击后提示"原文链接已失效，可尝试搜索政策名称获取最新信息"）
- 更新机制：每次我搜索更新数据时，批量验证URL有效性

## 七、技术栈与文件结构

```
长护险监测平台/
├── index.html              ← 独立版（内联CSS+JS+数据，双击打开）
├── app.py                  ← Flask后端（部署到Render用）
├── requirements.txt
├── Procfile
├── render.yaml
├── data/
│   ├── provinces.json      ← 28省份 + 城市列表 + 状态 + 关键数据
│   ├── policies.json       ← 政策文件（4类+2层级+URL）
│   ├── news.json
│   ├── supervision.json
│   ├── timeline.json
│   └── alerts.json
├── static/
│   ├── css/style.css
│   └── js/app.js
├── templates/index.html    ← Flask模板（动态引用static/）
└── DESIGN.md               ← 本文档
```

### 独立版 vs 部署版

| 版本 | 使用方式 | 数据更新 |
|------|----------|----------|
| **独立版** `index.html` | 双击打开，无需服务器 | 我来更新数据 → 重新生成文件 |
| **部署版** Flask | 部署到Render，获得在线网址 | 更新data/JSON → 重新部署 |
